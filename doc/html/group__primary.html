◆ TM_BEGIN

#define TM_BEGIN ( )

Prepare the procedure for the TM macros.

TM_BEGIN() creates the initial code enabling the macros to execute correctly. It should be placed at/near the start of the procedure. No TM_YIELD*() calls can come before it

◆ TM_END

#define TM_END ( )

Close out the TM macro component.

TM_END() balances TM_BEGIN(), closing the control structures TM_BEGIN() had created. It should be placed near the bottom of the procedure. No TM_YIELD*() calls can come after it.

◆ TM_YIELD

#define TM_YIELD ( n)

Yield and return to the next statement.

Perform a yield() operation. Upon return to this task, execution will continue at the next statement.

Parameters
n– an integer label. The label should be unique for all of the TM_YIELD*() routines in this task.

Yield with a delay and return to the next statement.
